# 1. 聚类任务 (Clustering Task)

聚类（Clustering）是无监督学习（unsupervised learning）中的一项核心任务。其根本目标是将一个数据集中的对象分组成若干个簇（cluster），使得同一个簇内的对象彼此之间具有较高的相似性，而不同簇的对象之间则差异较大。与监督学习不同，聚类任务事先并不知道任何样本的标记信息，因此它在探索和发现数据内在结构与规律方面扮演着至关重要的角色。

形式化地说，给定一个包含 m 个样本的数据集 $D = \{x_1, x_2, \dots, x_m\}$，聚类任务旨在将数据集 D 划分成 k 个互不相交的簇 $\{C_l | l=1, \dots, k\}$，满足 $\bigcup_{l=1}^{k} C_l = D$ 且对于 $i \neq j$ 有 $C_i \cap C_j = \emptyset$。每个簇都对应着一个潜在的概念。

一旦通过聚类算法形成了簇，如何评估聚类结果的质量就成为了一个关键问题。因此，我们需要一套有效的性能度量方法来衡量聚类效果的好坏。

# 2. 性能度量 (Performance Measures)

对聚类结果进行性能评估，是判断聚类算法优劣和比较不同算法性能的基础。聚类的性能度量（performance measures）主要分为两大类：一类是借助外部参考信息进行评估的“外部指标”（external indices），另一类则是仅依赖数据自身信息进行评估的“内部指标”（internal indices）。

## 外部指标 (External Indices)

外部指标通过将聚类结果与一个已知的“真实”分类（ground-truth）进行比较来衡量其质量。假定聚类给出的簇划分为 $C$，参考的真实划分为$C^*$，数据集中包含 m 个样本。我们可以从数据集中任意抽取两个样本，根据它们在 $C$ 和 $C^*$中的关系，定义以下四种情况：

- a: 样本对在 $C$ 中属于同一簇，在 $C^*$ 中也属于同一类。
- b: 样本对在 $C$ 中属于同一簇，但在 $C^*$ 中属于不同类。
- c: 样本对在 $C$ 中属于不同簇，但在 $C^*$ 中属于同一类。
- d: 样本对在 $C$ 中属于不同簇，在 $C^*$ 中也属于不同类。

基于这四个值，可以导出多种常用的外部评价指标。

- **Jaccard Coefficient (JC)** Jaccard系数衡量了聚类结果与真实划分在同簇/同类样本对上的一致性。 $$JC = \frac{a}{a+b+c}$$
- **Fowlkes and Mallows Index (FMI)** FMI从几何平均的角度评估聚类结果的准确性。 $$FMI = \sqrt{\frac{a}{a+b} \cdot \frac{a}{a+c}}$$
- **Rand Index (RI)** Rand指数衡量了聚类结果在所有样本对上的判断正确率。由于样本对总数为 $m(m-1)/2$，该值等于 a+b+c+d。 $$RI = \frac{2(a+d)}{m(m-1)}$$
上述三个指标均在 $[0,1]$ 区间，值越大越好。
## 内部指标 (Internal Indices)

内部指标不依赖任何外部参考信息，仅根据聚类结果自身的特性来评估其质量。其核心思想是，好的聚类结果应该具有高的“簇内相似度”和低的“簇间相似度”。

- **Davies-Bouldin Index (DBI)** DBI计算了每个簇与其最相似簇之间的相似度，并取其平均值。该指标越**小**，意味着簇内样本越紧密，且簇间分离度越好。 $$DBI = \frac{1}{k} \sum_{i=1}^{k} \max_{j \neq i} \left( \frac{\text{avg}(C_i) + \text{avg}(C_j)}{\text{dist}(c_i, c_j)} \right)$$ 其中，$avg(C_i)$ 是簇 $C_i$ 内任意两个样本之间的平均距离，$dist(c_i, c_j)$ 是簇 $C_i$ 和 $C_j$ 的质心 (均值向量) 之间的距离。**DBI值越小，表示聚类效果越好**。
- **Dunn Index (DI)** DI衡量了任意两个簇之间最短距离与任意簇内最大距离的比值。该指标越**大**，意味着簇间距离远，且簇内距离近。 $$DI = \frac{\min_{1 \le i < j \le k} \text{dist}(C_i, C_j)}{\max_{1 \le l \le k} \text{diam}(C_l)}$$ 其中 $diam(C)$ 是簇 C 的直径，即簇内任意两点间的最远距离。**DI值越大，表示聚类效果越好**。

无论是外部指标还是内部指标，其计算都离不开对样本间相似性或距离的定义。接下来我们将探讨如何计算样本间的距离。

# 3. 距离计算 (Distance Calculation)

距离度量（distance measure）是聚类算法的基石，因为它直接决定了如何量化样本之间的“相似性”。选择合适的距离度量至关重要，它将直接影响簇的形状和最终的聚类结果。样本属性的类型不同，适用的距离计算方法也不同。

## 有序属性 (Ordered Attributes)

对于具有数值顺序或大小关系的属性，例如身高、年龄等，最常用的距离是“闵可夫斯基距离”（Minkowski distance）。对于两个 d 维样本 $x_i$ 和 $x_j$，其闵可夫斯基距离定义为： $$\text{dist}_{\text{mink}}(x_i, x_j) = \left( \sum_{u=1}^{d} |x_{iu} - x_{ju}|^p \right)^{1/p}$$ 这个通用公式在不同参数 p 下会演变为几种常见的距离度量：

- 当 p=2 时，为**欧几里得距离**（Euclidean distance），也是最常用的距离度量。
- 当 p=1 时，为**曼哈顿距离**（Manhattan distance），也称“城市街区距离”。

## 无序属性 (Unordered Attributes)

对于不具有序关系的类别属性，例如“瓜类”的取值“西瓜”、“南瓜”等，闵可夫斯基距离不再适用。此时，可以采用“值差异度量”（Value Difference Metric, VDM）。VDM的威力在于它将属性值之间的距离建立在它们与最终类别标签的关联上。其核心思想是，如果两个属性值在引导样本分类上的表现越相似，它们的距离就越近。其距离公式为： $$\text{VDM}_p(a, b) = \sum_{i=1}^{C} \left| \frac{m_{u,a,i}}{m_{u,a}} - \frac{m_{u,b,i}}{m_{u,b}} \right|^p$$ 其中，$m_{u,a}$ 表示在属性 u 上取值为 a 的样本数，$m_{u,a,i}$ 表示在这些样本中第 i 类的样本数。此公式通过计算两个属性值 a 和 b 在所有类别上的**条件概率之差**来度量它们的距离。

## 混合属性 (Mixed Attributes)

在实际应用中，数据集往往同时包含有序属性和无序属性。此时，可以通过将闵可夫斯基距离与VDM结合起来，形成一个混合距离度量。通常的做法是对不同类型的属性分别计算距离，然后进行加权组合。

## 非度量距离 (Non-metric Distance)

在某些特殊情况下，我们使用的“距离”可能不满足数学上的所有度量性质，例如不满足三角不等式（即两边之和大于第三边）。这种距离被称为非度量距离。

有了计算样本间距离的方法，我们就可以开始探索具体的聚类算法了。首先介绍的是基于原型的聚类方法。

# 4. 原型聚类 (Prototype-based Clustering)

原型聚类（Prototype-based clustering），也称为“基于原型的聚类”，是一类假设每个簇都由一个原型（prototype）来代表的算法。这个原型可以是簇内样本的均值（即质心），也可以是簇中某个真实存在的样本点（即中心点）。聚类的过程就是将样本划分到其最近的原型所代表的簇中。

## k-means 算法

k-means 算法是原型聚类中最负盛名、应用最广泛的算法。它假设簇的结构是凸的，并试图找到 k 个簇，使得所有样本到其所属簇的质心（centroid）的平方误差之和最小。这个目标函数通常被称为“平方误差和”（Sum of Squared Errors, SSE）： $$E = \sum_{i=1}^{k} \sum_{x \in C_i} |x - \mu_i|_2^2$$ 其中$\mu_i$是第 i 个簇 $C_i$ 的均值向量（质心）。

k-means 算法的执行过程是一个迭代优化的过程：

1. **初始化**：从数据集中随机选择 k 个样本作为初始的簇质心。
2. **分配**：对数据集中的每个样本，计算其与 k 个质心的距离，并将其分配给距离最近的质心所对应的簇。
3. **更新**：对每个簇，重新计算其质心（即簇内所有样本的均值）。
4. **重复**：重复步骤 2 和 3，直到质心不再发生变化或达到最大迭代次数。

**主要局限性**：k-means 算法虽然简单高效，但其性能高度依赖于初始质心的选择，不同的初始值可能导致收敛到不同的局部最优解。

## 学习向量量化 (Learning Vector Quantization - LVQ)

学习向量量化（LVQ）是另一种基于原型的聚类算法。与 k-means 不同，LVQ 算法在聚类过程中利用了样本的标记信息（即监督信息）来辅助簇的划分。这使得LVQ特别适用于那些我们希望根据少量标记数据来优化簇边界的场景，从而有效地微调原型向量以对齐已知的类别。

LVQ 算法的步骤如下：

1. **初始化**：为每个类别初始化一组原型向量，并赋予它们相应的类别标记。
2. **迭代**：从训练集中随机选择一个样本，找到与其距离最近的原型向量。
3. **更新**：
    - 如果该原型向量的标记与样本标记**相同**，则将原型向量向样本方向拉近。
    - 如果该原型向量的标记与样本标记**不同**，则将原型向量向远离样本的方向推开。
4. **重复**：重复步骤 2 和 3 直到满足停止条件。

## 高斯混合聚类 (Gaussian Mixture Clustering)

高斯混合聚类采用了一种完全不同的视角，它基于概率模型进行聚类。该方法假设整个数据集是由若干个高斯分布（Gaussian distributions）混合生成的，其中每一个高斯分布就对应一个簇。

一个高斯混合模型由以下参数定义：

- **混合系数** ($\alpha_i$): 代表第 i 个高斯分布（簇）被选择的概率。
- **均值向量** ($\mu_i$): 第 i 个高斯分布的中心。
- **协方差矩阵** ($\Sigma_i$): 第 i 个高斯分布的形状和延展方向。

由于我们无法直接观测到这些参数，通常采用**期望最大化**（Expectation-Maximization, EM）算法来迭代求解，以找到参数的最大似然估计。EM算法交替执行两步：E步（Expectation）计算每个样本属于各个簇的后验概率，M步（Maximization）根据E步的结果更新模型参数。

虽然原型聚类方法很高效，但它们依赖于到中心点的距离，这隐含地假设了簇是凸形的（例如球状）。为了发现真实世界数据中常见的任意形状的簇，如新月形或环环相扣的结构，我们需要转向一种完全不同的范式：密度聚类。

# 5. 密度聚类 (Density-based Clustering)

密度聚类（Density-based clustering）是一类基于数据点密度的聚类方法。与原型聚类不同，它不假设簇具有任何特定形状。其核心思想是，一个簇是数据空间中由高密度区域组成的、并由低密度区域分隔开的样本集合。这种方法能够发现任意形状的簇，并且能有效地识别和处理噪声点。

## DBSCAN

DBSCAN (Density-Based Spatial Clustering of Applications with Noise) 是最具代表性的密度聚类算法。它依赖于几个核心概念来定义簇：

- **ε-邻域 (ε-neighborhood)**: 对于一个给定的样本点，其 ε-邻域是指数据集中与该点距离不超过 ε 的所有样本的集合。
- **核心对象 (Core object)**: 如果一个样本点的 ε-邻域内至少包含 `MinPts` 个样本（包括其自身），则该点被称为核心对象。
- **密度直达 (Directly density-reachable)**: 如果样本点 q 位于核心对象 p 的 ε-邻域内，则称 q 从 p 密度直达。
- **密度可达 (Density-reachable)**: 如果存在一个样本链 $p_1, p_2, \dots, p_n$，其中 $p_1=p$，且 $p_{i+1}$ 从 $p_i$ 密度直达，那么称 $p_n$ 从 $p$ 密度可达。
- **密度相连 (Density-connected)**: 如果存在一个核心对象 o，使得样本点 p 和 q 都从 o 密度可达，则称 p 和 q 是密度相连的。

基于这些概念，DBSCAN中的一个簇被形式化地定义为一个非空样本子集 C，它满足：对于任意点 p, q，如果 $p \in C$ 是一个核心对象且 q 从 p 密度可达，则 $q \in C$（最大性）；且 C 中任意两点 p, q 都是密度相连的（连接性）。

DBSCAN 的聚类过程如下：

1. 从数据集中任意选择一个未被访问的样本点。
2. 检查该点是否为核心对象。如果是，则以该点为种子创建一个新簇，并将所有从它密度可达的样本点都加入到这个簇中。
3. 如果该点不是核心对象，则暂时将其标记为噪声点。
4. 重复上述过程，直到数据集中所有点都被访问过。

除了上述的扁平化划分方法，还有一种能够揭示数据多层次结构的聚类方法，即层次聚类。

# 6. 层次聚类 (Hierarchical Clustering)

层次聚类（Hierarchical clustering）旨在构建一个嵌套的簇的层级结构。这种结构通常被可视化为一种树状图，称为“树状图”（dendrogram），它展示了簇在不同层次上的合并或分裂关系。层次聚类主要有两种策略：

- **自底向上（Agglomerative）**: 从将每个样本视为一个独立的簇开始，然后迭代地合并最接近的两个簇，直到所有样本都合并成一个簇。
- **自顶向下（Divisive）**: 从将所有样本视为一个大簇开始，然后迭代地分裂簇，直到每个样本都自成一簇。

## AGNES

AGNES (Agglomerative Nesting) 是一种采用自底向上策略的经典层次聚类算法。其过程非常直观：

1. **初始化**：将数据集中的每个样本都看作一个初始簇。
2. **迭代合并**：在每一步中，找到距离最近的两个簇并将它们合并成一个新簇。
3. **重复**：重复步骤 2，直到只剩下一个簇。

在合并过程中，关键在于如何定义两个簇之间的距离。不同的“链接标准”（linkage criteria）会导致不同的聚类结果。

- **最小距离 (Single-linkage)** 定义为两个簇中最近的两个样本点之间的距离（单链接）： $$d_{\text{min}}(C_i, C_j) = \min_{x \in C_i, z \in C_j} \text{dist}(x, z)，$$其中 x 是簇 $C_i$ 中的一个样本，z 是簇 $C_j$中的一个样本。
- **最大距离 (Complete-linkage)** 定义为两个簇中最远的两个样本点之间的距离（全连接）： $$d_{\text{max}}(C_i, C_j) = \max_{x \in C_i, z \in C_j} \text{dist}(x, z)$$，其中 x 是簇 $C_i$ 中的一个样本，z 是簇 $C_j$ 中的一个样本。
- **平均距离 (Average-linkage)** 定义为两个簇中所有样本对之间距离的平均值（均链接）： $$d_{\text{avg}}(C_i, C_j) = \frac{1}{|C_i||C_j|} \sum_{x \in C_i} \sum_{z \in C_j} \text{dist}(x, z)$$，其中 x 是簇 $C_i$ 中的一个样本，z 是簇 $C_j$ 中的一个样本。

与产生单一划分的“扁平”聚类方法相比，层次聚类提供了一种对数据结构更深入、更详细的审视方式。
